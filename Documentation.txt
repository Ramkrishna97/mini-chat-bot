============  PROOJECT  WORKFLOW  =================
Controller :
    REST : LLM controller ->("/api/llm") uses POST ("/query"), GET ("/query") methods to receive data as query and send response
            as LLM response to UI.   It also has a option to choose model selector.
            Uses LLMService and modelManagerService to process data and send response.
    MVC : Web Controller -> Handles the UI using thymeleaf. on base URL("/") it returns index.html. index() -> binds available models , base URL , server port using Model to modelManagerService.
            askQuestion() -> processes query by using llmService ,then binds data(model,query,URl,)to Model. and return index.html.
Service :
    LLMService : Uses ollamaService and modelManagerService .
        ProcessQuery : Check available models from modelManager Service , if model is not null and not exhausted, then it generates response using 'tryModel' method.
        tryModel : Uses ollamaService to generate response from input(query ,model) .
        isErrorResponse : checks if response has any "ERROR" message, then it tries to switch from any other available models.

    Ollama Service : uses OLLAMA_URL(fixed port for local- 11434 ),httpClient & objectMapper.
                     generateResponse() -> generates a requestBody from given model and query ,creates http request using uri,header,requestBody. Also, creates response by binding request to httpClient's send menthod.

    ModelManagerService: has a list of all available LLm models. setupModels() -> uses Process/Runtime.exec to run ollama-version , pull models ,command.
Model : Saves query and model data.
Config :
    AppStartupRunner :Uses modelManagerService.setupModels to set the models.
